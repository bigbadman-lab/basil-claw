# Basil Clawthorne 🦞 — System Architecture

Basil Clawthorne is a retrieval-grounded AI reply agent with a canon-driven Victorian parliamentary persona.

The system is designed to:

- Ground replies in structured policy material
- Avoid hallucination via vector retrieval
- Maintain a consistent rhetorical identity (Basil canon)
- Operate as an automated X (Twitter) mention-response agent

## High-Level Overview

Basil uses hybrid retrieval:

- Vector similarity (pgvector, cosine distance)
- Model filtering (embedding model consistency)
- Source-type weighting (summary > url > canon)
- Diversity filtering (max 2 chunks per source)
- Named-entity lexical anchoring (for high-precision stance resolution)

The final reply is generated by a chat model constrained by:

- Canon persona
- Retrieved source chunks
- Strict stylistic rules

## Core Components

### 1. Knowledge Store (Postgres + pgvector)

Tables:

- **sources** — metadata for each ingested source
- **chunks** — text fragments derived from sources
- **embeddings** — vector representations of chunks
- **x_mentions** — incoming mentions (planned loop)
- **x_replies** — reply logs
- **x_cursor** — last processed mention id

### 2. Ingestion Pipeline

Supports:

- Canon files
- Structured summary markdown
- Policy URLs

Process:

1. Load source
2. Chunk into semantic blocks
3. Generate embeddings
4. Store in Postgres
5. Attach embedding model metadata

### 3. Retrieval Engine

Given a query:

1. Generate embedding
2. Retrieve top-K nearest chunks (cosine distance)
3. Filter by embedding model
4. Apply source weighting
5. Apply named-entity lexical anchor (if triggered)
6. Enforce diversity (max 2 per source)
7. Reject weak matches via thresholding

### 4. Reply Generation

Inputs:

- Canon text
- Retrieved context chunks
- User tweet
- Intent classification

Constraints:

- 1–2 sentences
- No hallucinated facts
- No links unless asked
- No hashtags
- Canon voice adherence

## Design Philosophy

Basil is not a general chatbot.

It is:

- Persona-bound
- Policy-grounded
- Retrieval-constrained
- Architected for auditability

All factual grounding must originate from retrieved sources.
